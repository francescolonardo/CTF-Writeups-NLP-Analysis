<h3>Solution</h3>

<p>The website contains a list of books with two actions available:
* <em>an excerpt</em> to read an excerpt of the book;
* <em>read online</em> which opens a link from another domain, not related to the challenge.</p>

<p>Analyzing the HTML source of the page a <a href="treasury.js">treasury.js</a> file can be found; it is interesting to understand performed calls.</p>

<p>The read online button simply opens a window redirecting to the link specified for the book.</p>

<p>A URL like the following will reveal that web application is vulnerable to SQL injection, because the result of the book with <code>id=1</code> will be printed.</p>

<p><code>
https://poems.asisctf.com/books.php?type=excerpt&amp;id=0%27%20or%20id=%271
</code></p>
