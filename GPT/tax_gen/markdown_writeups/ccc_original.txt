<h2>Exploit</h2>

<p>First, in the website source, you can see that there's a route to login (<code>/login</code>) and a route to get the names of the admins (<code>/adminNames</code>). So, when you try to visit <code>/login</code>, you see a form. Post that and check the response on the Network Tab in your browser. You can see there's a header called <code>token</code> which stores a JWT. You can decode the token (maybe on <a href="https://jwt.io">jwt.io</a>) to see that the payload is of this format (I typed username: asd, password: asd):</p>

<p><code>
{
  "username": "nfq",
  "password": "nfq",
  "admin": "snyfr",
  "iat": 1593506966
}
</code></p>

<p>We can see that the <code>username</code> changed from <code>asd</code> to <code>nfq</code>. So this hints that this could be <code>rot13</code>. When you apply the <code>rot13</code> cipher on the admin value <code>snyfr</code>, you see that it returns a string <code>false</code>. So, we can change this to a rot13 encrypted string for <code>true</code>, but to do that, we need the JWT secret.
<br /></p>
