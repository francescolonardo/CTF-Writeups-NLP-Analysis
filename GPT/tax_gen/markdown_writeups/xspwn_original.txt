<li>There is a Use After Free (UAF) when deleting a node. The <code>HEAD</code> is never updated if the first index is removed.</li>
</ol>

<p><code>c
int deleteNameAtIndex(int index) {
    JARED *curr = HEAD;
    JARED *prev = NULL;
    int i = 0;
    while (curr != NULL) {
        if (i == index) {
            prev-&gt;next = curr-&gt;next;
            free(curr);
            return 0;
        }
        prev = curr;
        curr = curr-&gt;next;
        i++;
    }
    return 1;
}
</code></p>

<p>We could make use of this by first "inserting" our XSS payload (which will fail but write the payload to memory nonetheless), then overwriting the <code>HEAD-&gt;name</code> to the address of this XSS payload. <code>getNameAtIndex(0)</code> will then return the XSS payload when displaying the table!</p>

<p>For instance, if the first item we insert is our XSS payload, then the newly inserted item after the head node is deleted should have the name <code>\x20\x0e\x50\x00</code>.</p>

<p>We could set a breakpoint after the <code>malloc</code> call and before the <code>strcpy</code> call. Notice that <code>$var8</code> is set immediately after <code>call $dlmalloc</code> - this would be the pointer to the newly allocated memory where the payload will be stored.</p>
