<h3>Port 10010 [Web]</h3>

<p>When we log into the application, we can see the following data in the page source. There seems to be a <code>role</code> attribute that we need to change, in order to escalate our privileges.</p>

<p>```markup</p>

<script>
    var current_account = {
    "id":3,
    "username":"username",
    "password":"password",
    "role":"user",
    "created_at":"2021-12-04T05:12:11.986Z",
    "updated_at":"2021-12-04T05:12:11.986Z"};
</script>

<p>```</p>

<p>Taking a closer look at the registration fields, we see that we are submitting an <code>account</code> object with the <code>username</code> and <code>password</code> attributes.</p>

<p>```markup</p>

<div>
  <label for="account_username">Username</label>
  <input type="text" name="account[username]" id="account_username" />
</div>

<div>
  <label for="account_password">Password</label>
  <input type="password" name="account[password]" id="account_password" />
</div>

<div>
  <input type="submit" name="commit" value="Register" class="btn btn-primary" data-disable-with="Register" />
</div>

<p>```</p>

<p>Submitting with <code>account[role] = admin</code> changes our <code>role</code>, granting us access to <code>/admin</code>.</p>
