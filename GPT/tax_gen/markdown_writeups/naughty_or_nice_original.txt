<h2>Solution</h2>

<ol>
<li><p>Looking at the source code we see that in the <code>challenge/helpers/JWTHelper.js</code> file, both the "RS256" and "HS256" algorithms are allowed for decryption. This is CVE-2016-5431/CVE-2016-10555.</p></li>
<li><p>The HS256 algorithm is symmetric, which means it uses the sane secret key to sign and verify each message. The RS256 algorithm is asymmetric, which means it uses a private key to sign the message and a public key for verification. However, if we change the algorithm from RS256 to HS256, the backend code will use the public key as the symmetric secret key. In other words, the HS256 algorithm will be used to verify the signature with the public key as the HS256 secret key. We know the public key so we can, in theory, easily modify the JWT and sign it.</p></li>
<li><p>However, before we can do this we need to get the JWT token and get the public key. So, make an account on the website, open up your browser's cookie page in the developer tools, and copy the <code>session</code> cookie. <li><p>Back in the source code, if we look at the <code>challenge/routes/index.js</code> file we see that the <code>/</code> endpoint uses the <code>CardHelper</code> class to generate the card with a list of the elf's names. Since we are now an admin, we can edit the names of these elfs, and thus we control this value. Looking at <code>challenge/helpers/CardHelper.js</code>, we notice that the <code>nunjucks</code> templating engine is used and that we can contol what is passed to a <code>nunjucks</code> template. <li><p>Searching for "nunjucks ssti" reveals <a href="http://disse.cting.org/2016/08/02/2016-08-02-sandbox-break-out-nunjucks-template-engine">this great guide</a> about breaking out of the <code>nunjucks</code> templating engine. We can run <code>{{range.constructor("return global.process.mainModule.require('child_process').execSync('cat /flag*')")()}}</code> to print the contents of the <code>/flag</code> file. We determined that this is where the flag is by looking at the <code>Dockerfile</code> in the challenge ZIP. The <code>flag</code> file is copied to <code>/flag</code> within the container.</p></li>
<li><p>So, in the admin dashboard, we copy and paste out payload <code>{{range.constructor("return global.process.mainModule.require('child_process').execSync('cat /flag*')")()}}</code> into one off the elf's name field, submit the changes, and then we navigate back to the home page, <code>/</code>, and there's the flag. You can view the source of this page to make copy and pasting the flag easier.</p></li>
