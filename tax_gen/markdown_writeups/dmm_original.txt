<h2>Solution</h2>

<p>The <em>SSTI</em> in the URL is a hint to <em>Server-Side Template Injection</em> vulnerabilities.</p>

<p><code>http://ssti.darkarmy.xyz/?url={{7*7}}</code> will return the following.</p>

<p>```html</p>

<div style = "text-align:center">
<h1>Oops! 49 is 404 Not Found.</h1>
</div>

<p>```</p>

<p><code>http://ssti.darkarmy.xyz/?url=[]</code> will return</p>

<p><code>html
ohhhhhhhhhhhh noooooo0ooooo !! :) [
</code></p>

<p>The same happens for chars: <code>%</code>, <code>[</code>, <code>]</code>, <code>"</code>, <code>+</code>, <code>_</code> and for words: <code>self</code>, <code>request</code>, <code>join</code>, <code>config</code>.</p>

<p>So the template engine is probably <em>Jinja2</em>, but you have to face some restrictions.</p>
