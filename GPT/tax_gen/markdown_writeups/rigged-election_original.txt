<p>The voting procedure requires that a sort of challenge is returned by the server after the first request, i.e. <code>xhttp.open ("GET", "/vote.php?g=1", false);</code>, and then the client has to find a string which produces a MD5 hash with the first characters equals to the challenge. The challenge is different each time.</p>

<p>To reduce the time of finding correct strings, a smart voter can be implemented, using a cache and storing all failed attempts. With this trick, after ~14M  requests all strings will be found inside the cache, reducing the time of generation.</p>
