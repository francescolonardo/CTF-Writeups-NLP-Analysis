<p>Even if the <code>$name</code> variable is concatenated inside a MySQL comment, the code is still vulnerable to SQL injection, because MySQL can execute query portions inside comments.</p>

<p>From <a href="https://www.owasp.org/index.php/Testing_for_MySQL">https://www.owasp.org/index.php/Testing<em>for</em>MySQL</a>:</p>

<blockquote>
  <p>Fingerprinting MySQL</p>
  
  <p>Of course, the first thing to know is if there's MySQL DBMS as a back end database. MySQL server has a feature that is used to let other DBMS ignore a clause in MySQL dialect. When a comment block ('/<em>*/') contains an exclamation mark ('/</em>! sql here*/') it is interpreted by MySQL, and is considered as a normal comment block by other DBMS as explained in MySQL manual.</p>
  
  <p>Example:</p>
  
  <p>1 /*! and 1=0 */</p>
  
  <p>Result Expected:</p>
  
  <p>If MySQL is present, the clause inside the comment block will be interpreted.</p>
</blockquote>
