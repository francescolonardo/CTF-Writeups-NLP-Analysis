<h1>Pwn2Win CTF 2021 [web] Illusion writeup</h1>

<h2>Challenge</h2>

<p>Laura just found a website used for monitoring security mechanisms on Rhiza's state and is planning to hack into it to forge the status of these security services. After that she will desactivate these security resources without alerting government agents. Your goal is to get into the server to change the monitoring service behavior.</p>

<p>Server: <code>nc illusion.pwn2win.party 1337</code></p>

<h2>Solution</h2>

<p>This is a simple web application that stores the status of services.</p>

<p>You have to execute <code>./readflag</code> to get the flag.</p>

<p>There is an API that can modify the status of services.</p>

<p>```javascript
// API
app.post("/change_status", (req, res) =&gt; {</p>

<pre><code>let patch = []

Object.entries(req.body).forEach(([service, status]) =&gt; {

    if (service === "status"){
        res.status(400).end("Cannot change all services status")
        return
    }

    patch.push({
        "op": "replace",
        "path": "/" + service,
        "value": status
    })
});

jsonpatch.applyPatch(services, patch)

if ("offline" in Object.values(services)){
    services.status = "offline"
}

res.json(services)
</code></pre>

<p>})
```</p>

<p><code>
curl -i -X POST \
  http://localhost:1337/change_status \
  --header "Content-type: application/json" \
  --data '{"cameras": "online","doors": "online","dome": "online","turrets": "offline"}'
</code></p>

<p>Now, at first glance, these do not appear to be vulnerable.</p>

<p>And all the npm packages used seem to be up to date.</p>

<p><code>json
{
  "dependencies": {
    "ejs": "^3.1.6",
    "express": "^4.17.1",
    "express-basic-auth": "^1.2.0",
    "fast-json-patch": "^3.0.0-1"
  }
}
</code></p>

<p>However, if you check the package <a href="https://github.com/Starcounter-Jack/JSON-Patch">fast-json-patch</a>, you will find a Prototype Pollution vulnerability that is unfixed.</p>

<p><a href="https://github.com/Starcounter-Jack/JSON-Patch/pull/262">Security Fix for Prototype Pollution - Starcounter-Jack/JSON-Patch</a></p>

<p>This web application uses <code>ejs</code>, which is known to be capable of RCE by Prototype Pollution.</p>

<p><a href="https://github.com/mde/ejs/issues/451">Unrestricted render option may lead to a RCE vulnerability - mde/ejs</a></p>
