<h1>Challenge Title: Web-1</h1>

<h2>Description</h2>

<p>May the cURL be with you xD</p>

<p>Flag format: darkCTF{FLAG}</p>

<p><a href="http://web1.darkarmy.xyz/">Link</a></p>

<h2>Solution</h2>

<p>On opening the link the site says <em>to login move to /login by post and send json request</em> and as the description say to use cURL so lets start with it </p>

<p><code>
curl -X POST http://web1.darkarmy.xyz/ -H 'content-type: application/json' --data '{"user":"karma","pass":"karma"}'
</code>
In return, we get a jwt token</p>

<p><img src="https://github.com/karma9874/CTF-Writeups/blob/master/Dark-PreCTF/Images/login1.JPG" alt="alt text" /></p>

<p>Used <a href="https://jwt.io/">jwt.io</a> to decode that token which had payload like this <code>{"user":"guest"}</code>.So what now?</p>

<p>Always check robots.txt of all web challs in any CTF because sometime it may have some data which can be useful.</p>

<p>Just like in this chall accessing <code>/robots.txt</code> we get <code>/flag</code> page</p>

<p><img src="https://github.com/karma9874/CTF-Writeups/blob/master/Dark-PreCTF/Images/robots1.JPG" alt="alt text" /></p>

<p>We cant make a GET request to <code>/flag</code> so maybe POST?</p>

<p>On making a POST request to <code>/flag</code> we get something like this <code>gimme token and I will give you the flag</code></p>

<p>On passing the jwt token which we got earlier we get a response like this <code>Not the admin, no flag for you</code>. So it needs admin token, well we can edit the token to <code>{"user":"admin"}</code> using <a href="https://jwt.io/">jwt.io</a> and passing that token to <code>/flag</code> we get the flag</p>

<p><img src="https://github.com/karma9874/CTF-Writeups/blob/master/Dark-PreCTF/Images/flag1.JPG" alt="alt text" /></p>

<p>flag -> <code>darkCTF{345y_p345y_JWTs}</code></p>

<h2>What was the Bug?</h2>

<p>In this chall I was just checking the decoded value to have value of <code>user</code> to be <code>admin</code> (<code>{"user":"admin"}</code>) to show to the flag thats it, the code was not checking for any signature verification</p>
